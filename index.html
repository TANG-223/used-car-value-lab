How it works

A simple model-to-interface workflow

The website does not train a model online. It reads prepared coefficients, accepts user inputs, and computes an estimate directly in the browser.

01

Data and preprocessing

The original project cleaned real used-car data, reduced outliers, and prepared variables for regression analysis.

02

Regression model

A linear model was built as the baseline, and a cubic model was explored for comparison.

03

Interactive estimator

The web interface turns the model into a visible tool that users can test, compare, and discuss.

Learn the market

Why these inputs matter

This section helps users — especially students and Chinese readers learning about the U.S. market — understand common used-car pricing logic.

Mileage

Higher mileage often means more wear and longer usage history, which usually reduces resale value.

Vehicle age

Older vehicles generally lose value because of depreciation, aging components, and changing buyer preference.

Condition

Cars in better condition often receive higher estimates because they may require fewer repairs.

Brand and trim

Some brands and trims hold value better because of reliability, features, and buyer demand.

About this project

A complete high-school-style research and web demo project

This site was upgraded from an existing single-file estimator into a more complete project website with bilingual support, preset vehicles, clearer results, and educational sections.

It is still intentionally lightweight: static HTML, CSS, and JavaScript, easy to deploy on GitHub Pages, and realistic for a student-built project.
